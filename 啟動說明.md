# 登革熱監測系統啟動說明

## 問題排除

如果遇到 `ERR_CONNECTION_REFUSED` 錯誤，請按照以下步驟檢查：

### 1. 確認分析資料已產生

```bash
python src/analyze_dengue.py
```

### 2. 檢查端口是否被佔用

```bash
netstat -ano | findstr :8080
```

如果端口被佔用，可以：
- 關閉佔用端口的程式
- 或修改 `website/app.py` 中的 port 為其他數字（如 8081）

### 3. 啟動伺服器

**方法一：使用測試腳本（推薦）**
```bash
python test_server.py
```

**方法二：使用啟動腳本**
```bash
python start_website.py
```

**方法三：直接啟動**
```bash
cd website
python app.py
```

### 4. 開啟瀏覽器

在瀏覽器中開啟：**http://localhost:8080** 或 **http://127.0.0.1:8080**

## 常見問題

### Q: 顯示 "端口已被佔用"
A: 修改 `website/app.py` 第 56 行，將 `port=8080` 改為其他數字（如 `port=8081`）

### Q: 顯示 "分析資料不存在"
A: 執行 `python src/analyze_dengue.py` 產生分析資料

### Q: 顯示 "Flask 未安裝"
A: 執行 `pip install flask` 安裝 Flask

### Q: 網頁可以開啟但沒有資料
A: 確認 `data/processed/dengue_analysis.json` 檔案存在且格式正確

## 測試連線

啟動伺服器後，應該會看到類似以下的訊息：
```
 * Running on http://127.0.0.1:8080
 * Debug mode: on
```

如果看到這個訊息，表示伺服器已成功啟動，可以在瀏覽器中開啟網址。
